1. Overview

The directory uses three main collection types: Directory – Dioceses (diocese), Directory – Parishes (parish), and Directory – Priests (priest). Each priest and parish is tied to exactly one diocese. A priest may be linked as vicar of at most one parish (one-to-one with the parish’s vicar field). All three types also relate to Tenant for multi-site isolation.

2. Entities & relations (logical model)

The following table summarizes who relates to whom in the Strapi schema (not the physical DB table names).

From Relation To Meaning
Parish manyToOne (required) Diocese Every parish belongs to one diocese. Inverse: Diocese.parishes (oneToMany).
Priest manyToOne (required) Diocese Every priest is allocated to one diocese. Inverse: Diocese.priests (oneToMany).
Parish oneToOne (optional) Priest (vicar) Current vicar of the parish. Inverse: Priest.parish (mappedBy: vicar).
Diocese, Parish, Priest manyToOne Tenant All directory entries for a site share the same tenant; filter APIs by tenant.tenantId.
Frontend use cases: List priests for a diocese → filter priests by diocese and tenant. List parishes for a diocese → filter parishes by diocese. Show where a priest is serving as vicar → populate=parish on priest (or populate=vicar on parish).

4. Strapi 5 storage (SQLite link tables)

On SQLite, Strapi 5 does not store relations as diocese_id columns on priests or parishes. It uses link tables named *_lnk.

Link tableColumns (conceptual)Purpose
priests_diocese_lnkpriest_id, diocese_id, priest_ordPriest ↔ Diocese
parishes_diocese_lnkparish_id, diocese_id, parish_ordParish ↔ Diocese
parishes_vicar_lnkparish_id, priest_idParish ↔ Vicar (Priest)
priests_tenant_lnkpriest_id, tenant_id, …Priest ↔ Tenant
parishes_tenant_lnkparish_id, tenant_id, …Parish ↔ Tenant
dioceses_tenant_lnkdiocese_id, tenant_id, …Diocese ↔ Tenant
Sample database note: A typical seeded dev database may have many rows in priests_diocese_lnk and parishes_diocese_lnk but zero rows in parishes_vicar_lnk until vicars are assigned in Admin. That is expected if imports only set diocese, not vicar.
5. Tenant scoping

Per project rules (.cursor/rules/multi-tenant-assignment.mdc): editors do not pick a tenant in the UI; tenant is auto-assigned from Editor Tenant Assignment. Public or frontend API calls should always filter by the programmatic tenant id:

Filter pattern: filters[tenant][tenantId][$eq]=<yourTenantId>

6. REST API examples (frontend)

Replace STRAPI_URL, YOUR_TENANT_ID, and slug/id as needed. Use populate to load nested diocese, parish, or vicar.

Verify / API — List priests in a diocese (with parish)
🔧 Customize Parameters:
curl -s "http://localhost:1337/api/priests?filters[tenant][tenantId][$eq]=tenant_demo_002&filters[diocese][slug][$eq]=diocese-of-calcutta&populate[0]=diocese&populate[1]=parish"
Windows single-line
curl -s "http://localhost:1337/api/priests?filters[tenant][tenantId][$eq]=tenant_demo_002&filters[diocese][slug][$eq]=diocese-of-calcutta&populate[0]=diocese&populate[1]=parish"
Verify / API — Parishes in a diocese (with vicar)
🔧 Customize Parameters:
curl -s "http://localhost:1337/api/parishes?filters[tenant][tenantId][$eq]=tenant_demo_002&filters[diocese][slug][$eq]=diocese-of-calcutta&populate[0]=diocese&populate[1]=vicar"
Windows single-line
curl -s "http://localhost:1337/api/parishes?filters[tenant][tenantId][$eq]=tenant_demo_002&filters[diocese][slug][$eq]=diocese-of-calcutta&populate[0]=diocese&populate[1]=vicar"
Permissions: Ensure the role used by the frontend (e.g. Public) has find / findOne on priest, parish, and diocese, with populate allowed for the relations you need.
7. Verifying data in the database

From the repository root, run the inspection script (uses better-sqlite3 from the project). It prints counts for link tables and sample joined rows.

Script: inspect-directory-relationships-sample.js

Setup / inspect script
🔧 Customize Parameters:
node scripts/inspect-directory-relationships-sample.js
Windows single-line
node scripts/inspect-directory-relationships-sample.js
What to expect: Non-zero counts for priests_diocese_lnk and parishes_diocese_lnk when data is imported. parishes_vicar_lnk may be 0 until vicars are set in Admin.

Raw SQL (sqlite3 CLI)

SELECT COUNT(*) FROM priests_diocese_lnk;
SELECT COUNT(*) FROM parishes_diocese_lnk;
SELECT COUNT(*) FROM parishes_vicar_lnk;

SELECT pr.name AS priest, d.name AS diocese
FROM priests pr
JOIN priests_diocese_lnk l ON l.priest_id = pr.id
JOIN dioceses d ON d.id = l.diocese_id
LIMIT 10;

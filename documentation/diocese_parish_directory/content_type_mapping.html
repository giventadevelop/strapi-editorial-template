Reference: directory.mosc.in. All 12 sections on the portal are covered by 6 Strapi content types (no extra types needed).

Why one “Directory – Entries” for 7 sections? On the live site, Institutions, Church Dignitaries, Working Committee, Managing Committee, Spiritual Organisations, Pilgrim Centres, and Seminaries all use the same URL pattern (/directories/?type=...) and the same data shape (name, address, email, phones, website, optional description/role). So one collection type with a directoryType enum is enough and matches the portal. Editors choose the type when creating an entry; the frontend filters by directoryType to show each section.

Summary: Home, Bishops, Dioceses, Parishes, Priests = 5 dedicated content types. The other 7 sections = 1 content type (Directory – Entries) with directoryType. No additional content types are required.
